# Project -

ВСІ Серйозні кроки Треба Документувати, створювай файл під нові завдання та запиши свої дії щодо встановлення та виконання завдання.
Документуй свої дії – легше буде у наступній сесії

Коли пишеш запит JSON, то завжди закриваються подвійні лапки.

## Правила (завжди дотримуватися):
- Відповідай коротко, без довгих пояснень
- Одна дія за раз
- Не читай файли, які не просили
- /compact якщо контекст > 20k

## Завдання тестування:
1. Відповісти на просте запитання
2. Прочитати файл
3. Виконати bash команду
4. Створити/змінити файл
5. Використовувати кілька інструментів поспіль

## ПРАВИЛА — суворо дотримуватися:
- НЕ запускати sub-agents та background tasks
- Одна дія за раз, чекати на підтвердження
- Максимум 3 tool calls на одне завдання
- Не читати файли, які не просили явно
- Короткі відповіді без пояснень
