## Перший реальний крок — Terraform для AWS

### Структура

```bash
mkdir -p ~/repo/terraform/aws
cd ~/repo/terraform/aws
```

### Terraform конфіг — EC2 t2.micro + Security Group

```bash
cat > main.tf << 'EOF'
terraform {
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
  }
  # Пізніше переведемо state в S3
}

provider "aws" {
  region = "eu-north-1"
}

# Security Group
resource "aws_security_group" "k3s" {
  name        = "k3s-sg"
  description = "K3s node security group"

  ingress {
    description = "SSH from home"
    from_port   = 22
    to_port     = 22
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]
  }

  ingress {
    description = "HTTP"
    from_port   = 80
    to_port     = 80
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]
  }

  ingress {
    description = "HTTPS"
    from_port   = 443
    to_port     = 443
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]
  }

  egress {
    from_port   = 0
    to_port     = 0
    protocol    = "-1"
    cidr_blocks = ["0.0.0.0/0"]
  }

  tags = { Name = "k3s-sg", Project = "SysOps-to-DevOps" }
}

# EC2 t2.micro — Free Tier
resource "aws_instance" "k3s" {
  ami           = "ami-075449515af5df0d1"  # Ubuntu 22.04 eu-north-1
  instance_type = "t2.micro"

  vpc_security_group_ids = [aws_security_group.k3s.id]

  root_block_device {
    volume_size = 20
    volume_type = "gp2"
  }

  tags = { Name = "k3s-node", Project = "SysOps-to-DevOps" }
}

# Elastic IP — щоб IP не змінювався
resource "aws_eip" "k3s" {
  instance = aws_instance.k3s.id
  domain   = "vpc"
  tags     = { Name = "k3s-eip" }
}

output "public_ip"  { value = aws_eip.k3s.public_ip }
output "instance_id" { value = aws_instance.k3s.id }
EOF
```

### Запускаємо

```bash
cd ~/repo/terraform/aws
terraform init
terraform plan
```

Покажи вивід `terraform plan` — перевіримо що буде створено перед `apply`!

⚠️ **t2.micro** — безкоштовно 750 год/місяць (перший рік). Elastic IP безкоштовний поки прив'язаний до запущеного інстансу.

## Terraform є на Windows — запускаємо звідти!

AWS CLI налаштований на Ubuntu, але Terraform можна запускати з Windows. Потрібно тільки налаштувати AWS credentials на Windows:

```powershell
# Windows PowerShell
aws configure
# AWS Access Key ID: AKIA_REDACTED
# AWS Secret Access Key: (твій secret)
# Default region: eu-north-1
# Default output format: json

aws sts get-caller-identity
```

Потім:

```bash
# Windows Git Bash
cd /j/SysOps_to_DevOps
mkdir -p terraform/aws
cd terraform/aws
```

Створи `main.tf` з вмістом що надав вище і запусти:

```bash
terraform init
terraform plan
```

```bash

sudo apt update && sudo apt install terraform -y

terraform version
```

### Крок 1 — Встановлення AWS CLI

```powershell
winget install --id Amazon.AWSCLI -e --accept-source-agreements --accept-package-agreements
```

**Результат:** `aws-cli/2.34.57 Python/3.14.5 Windows/11 exe/AMD64`

### Крок 2 — Налаштування credentials

Отримав з Ubuntu (`ssh devops-lab "cat ~/.aws/credentials"`) і записав через `aws configure set`:

```powershell
aws configure set region eu-north-1
aws configure set output json
aws configure set aws_access_key_id AKIA_REDACTED
aws configure set aws_secret_access_key SECRET_REDACTED
```

**Примітка:** AWS CLI v2 на Windows **не приймає** config/credentials файли, записані через `Out-File` (навіть з BOM). Тільки `aws configure set` працює коректно.

**Перевірка:**
```powershell
aws sts get-caller-identity
# {
#   "UserId": "AIDAQ2PVDVESTMQLTBTMC",
#   "Account": "056885487909",
#   "Arn": "arn:aws:iam::056885487909:user/devops-admin"
# }
```

### Крок 3 — Створення terraform/aws/main.tf

**Файл:** `terraform/aws/main.tf`

**Відмінності від оригіналу в .sonet_free.md:**
- Замість захардкодженого AMI `ami-075449515af5df0d1` використано `data "aws_ami" "ubuntu_2204"` (автоматично знаходить актуальний)
- AMI вибирається фільтром: `ubuntu/images/hvm-ssd/ubuntu-jammy-22.04-amd64-server-*`
- Owner: `099720109477` (Canonical)

**Структура:**
```hcl
provider "aws" { region = "eu-north-1" }

data "aws_ami" "ubuntu_2204" { ... }

resource "aws_security_group" "k3s" { ... }   # SSH 22, HTTP 80, HTTPS 443
resource "aws_instance" "k3s"      { ... }   # t2.micro, 20GB gp2
resource "aws_eip" "k3s"           { ... }   # Elastic IP
```

**⚠️ key_name не вказано** — не зможемо зайти по SSH. Потрібно додати перед `apply`:
```hcl
resource "aws_key_pair" "k3s" {
  key_name   = "k3s-key"
  public_key = file("~/.ssh/id_ed25519.pub")
}
```
